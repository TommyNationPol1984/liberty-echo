Screen-by-screen specs (interaction highlights)
Home Dashboard
Primary actions: Create Clone, Synthesize Text, Dubbing Project, Singing Project.

Safety panel: Consent status, watermark toggle (default ON), abuse filter status.

Quick-play: Demo clone that runs without uploading to show capabilities.

Create Clone Wizard
Step gating: Consent verification required before sample upload is accepted.

Live analyzer: After upload, show SNR, sample rate, recommended duration, and “fix” suggestions (re-record, noise reduction).

Fidelity estimate: Display expected fidelity badge (Zeroshot / Fewshot) and cost estimate.

Synthesize Text
Editor: SSML-aware editor with inline phoneme preview and normalization suggestions.

Controls: Voice selector, emotion presets, continuous sliders (pitch, rate, energy), advanced per-syllable emphasis.

Preview: Streaming playback with latency indicator; stems toggle (voice, breath, background).

Export: Format selector (wav 48k/24bit, wav 24k/16bit, mp3 192–256kbps, m4a 192kbps) and loudness normalization options.

Dubbing Project
Upload: Source audio/video + transcript or auto-transcribe.

Alignment view: Forced-alignment markers; editable syllable timing; strict/loose timing toggle.

Mapping rules: Formant preservation slider, timestretch limits (±5–8%), and A/B compare.

Singing Studio
Inputs: MIDI upload, lyrics editor with phoneme mapping, tempo map.

Pitch editor: Note-level F0 curves, vibrato depth/rate handles, breath placement markers.

Render: Separate stems export and mix-ready postFX chain.

Admin & Audit
Consent logs: Tamper-evident entries, download proof, revoke/purge actions.

Abuse dashboard: Blocked requests, flagged keywords, rate-limit metrics.

Developer Handoff & API contract stubs
API endpoints (stubs for devs)
POST /api/clone

Payload: { voice_name, consent_proof_id, samples: [file_refs], language_tags, region }

Response: { clone_id, fidelity_estimate, status }

POST /api/synthesize

Payload: { text_or_ssml, voice_id, emotion, intensity, pitch, rate, format, sample_rate }

Response: streaming audio URL or stream handle; job_id for async.

POST /api/dub

Payload: { source_audio_ref, transcript_ref, target_voice_id, timing: "strict"|"loose", formant_preserve }

Response: { job_id, preview_url }

POST /api/revoke

Payload: { clone_id, requester_id, reason }

Response: { status: "revoked", purge_job_id }

QA, Accessibility, and Security checklist (high level)
Accessibility

Keyboard-first navigation for all flows.

ARIA labels for dynamic regions (synthesis progress, errors).

Captions/transcripts for audio/video previews.

Security & Privacy

Encrypted uploads in transit and at rest.

Ephemeral raw audio storage until consent verified.

One-click purge that flags model retraining and storage purge.

Ethics & Safeguards

Watermarking default ON; public figure/minor detection with blocking UI.

Required labeling on exported files: “Synthetic audio generated by AI”.
