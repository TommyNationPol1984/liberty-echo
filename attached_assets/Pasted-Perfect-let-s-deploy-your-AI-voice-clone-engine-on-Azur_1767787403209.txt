🐳 STEP 1 — Push Your Images to Azure Container Registry
Create ACR
bash
az acr create --resource-group TommyVoiceRG \
  --name TommyVoiceRegistry \
  --sku Basic
Login
bash
az acr login --name TommyVoiceRegistry
Tag images
bash
docker tag tommy-voice-api TommyVoiceRegistry.azurecr.io/tommy-voice-api:latest
docker tag tommy-voice-worker TommyVoiceRegistry.azurecr.io/tommy-voice-worker:latest
Push
bash
docker push TommyVoiceRegistry.azurecr.io/tommy-voice-api:latest
docker push TommyVoiceRegistry.azurecr.io/tommy-voice-worker:latest
📦 STEP 2 — Create Azure Storage
Create storage account
bash
az storage account create \
  --name tommyvoicestorage \
  --resource-group TommyVoiceRG \
  --location centralus \
  --sku Standard_LRS
Create blob containers
bash
az storage container create --name uploads --account-name tommyvoicestorage
az storage container create --name outputs --account-name tommyvoicestorage
az storage container create --name voices --account-name tommyvoicestorage
You’ll update your code to use Azure Blob URLs for reading/writing audio.

🧠 STEP 3 — Deploy API to Azure Container Apps
Create environment
bash
az containerapp env create \
  --name TommyVoiceEnv \
  --resource-group TommyVoiceRG \
  --location centralus
Deploy API service
bash
az containerapp create \
  --name tommy-voice-api \
  --resource-group TommyVoiceRG \
  --environment TommyVoiceEnv \
  --image TommyVoiceRegistry.azurecr.io/tommy-voice-api:latest \
  --target-port 8000 \
  --ingress external \
  --registry-server TommyVoiceRegistry.azurecr.io \
  --cpu 1 --memory 2Gi \
  --env-vars STORAGE_ACCOUNT=tommyvoicestorage
This gives you a public HTTPS endpoint like:

Code
https://tommy-voice-api.centralus.azurecontainerapps.io
⚙️ STEP 4 — Deploy Worker Service
Workers don’t need public ingress.

bash
az containerapp create \
  --name tommy-voice-worker \
  --resource-group TommyVoiceRG \
  --environment TommyVoiceEnv \
  --image TommyVoiceRegistry.azurecr.io/tommy-voice-worker:latest \
  --ingress internal \
  --registry-server TommyVoiceRegistry.azurecr.io \
  --cpu 1 --memory 2Gi \
  --env-vars STORAGE_ACCOUNT=tommyvoicestorage
The worker will automatically process queued jobs.

🔐 STEP 5 — Connect API + Worker + Storage
Your API will:

Upload reference audio → Azure Blob

Store embeddings → Azure Blob

Generate TTS → save output → Azure Blob

Return signed URLs to the client

Use azure-storage-blob in Python:

python
from azure.storage.blob import BlobServiceClient

blob = BlobServiceClient.from_connection_string(os.getenv("AZURE_STORAGE_CONNECTION_STRING"))
container = blob.get_container_client("outputs")

container.upload_blob(f"{job_id}.wav", audio_bytes)
🌐 STEP 6 — Custom Domain + HTTPS
Add domain
bash
az containerapp hostname bind \
  --resource-group TommyVoiceRG \
  --name tommy-voice-api \
  --hostname voice.tommy-nation.com \
  --certificate TommyVoiceCert
Azure Container Apps automatically handles TLS via Azure Front Door or managed certs.

📈 STEP 7 — Scaling
API autoscaling
bash
az containerapp update \
  --name tommy-voice-api \
  --resource-group TommyVoiceRG \
  --scale-rule-name http-scaling \
  --scale-rule-type http \
  --scale-rule-http-concurrency 50 \
  --min-replicas 1 \
  --max-replicas 10
Worker autoscaling (queue depth)
Once you move to Azure Queue Storage:

bash
az containerapp update \
  --name tommy-voice-worker \
  --resource-group TommyVoiceRG \
  --scale-rule-name queue-scaling \
  --scale-rule-type azure-queue \
  --scale-rule-metadata queueName=tts-jobs \
  --min-replicas 1 \
  --max-replicas 10
