## 4. Data & training pipeline

### 4.1. Collect and prepare voice data

- **Input:** 30 seconds–10 minutes of clean audio (WAV, 16 kHz or 22.05 kHz, mono).  
- **Steps:**
  - **Normalize:** resample, normalize loudness, trim silence.  
  - **Segment:** split into 3–10 second clips.  
  - **Transcribe:** use Whisper or similar to get text for each clip.  
  - **Align format:** match the dataset format expected by F5‑TTS or OpenAudio (CSV/JSON with `path,text`).

Example dataset CSV:

```text
/path/to/audio_0001.wav|This is my real voice for cloning.
/path/to/audio_0002.wav|I use this engine to narrate my podcast.
```

---

### 4.2. Training / fine‑tuning (F5‑TTS‑style)

F5‑TTS is an open‑source TTS system you can clone and train locally.

**Install & setup (conceptual):**

```bash
git clone https://github.com/SWivid/F5-TTS.git
cd F5-TTS
pip install -r requirements.txt
# plus PyTorch with CUDA if available
```

**Train / fine‑tune:**

```bash
python train.py \
  --data_path /path/to/your/dataset \
  --output_dir /path/to/checkpoints \
  --epochs 10 \
  --batch_size 16
```

This gives you a **voice‑conditioned TTS model** that can synthesize speech in that voice.

## 5. Inference: text‑to‑speech & voice cloning

### 5.1. Basic TTS inference (Python)

Once trained, you load the model and generate audio from text:

```python
from f5_tts.inference import load_model, tts
import soundfile as sf

model = load_model("/path/to/checkpoints/best_model.pt")

def synthesize(text: str, speaker_id: str, out_path: str):
    audio, sr = tts(
        model=model,
        text=text,
        speaker_id=speaker_id,  # or reference embedding
        language="en",
        style="neutral"
    )
    sf.write(out_path, audio, sr)
```

You can adapt this pattern to OpenAudio’s API if you choose that engine instead.

---

### 5.2. Voice cloning from reference audio

Two approaches:

1. **Embedding‑based cloning (few‑shot):**  
   - Extract a speaker embedding from 30s–10 min of reference audio.  
   - Pass that embedding into the TTS model at inference time.  

2. **Fine‑tuning per voice:**  
   - Train a small adapter or fine‑tune the model on that speaker’s dataset.  

Conceptual code for embedding‑based cloning:

```python
from my_speaker_encoder import encode_speaker
from f5_tts.inference import tts

speaker_embedding = encode_speaker("/path/to/reference.wav")

def clone_and_speak(text: str, out_path: str):
    audio, sr = tts(
        model=model,
        text=text,
        speaker_embedding=speaker_embedding,
        language="en",
        style="narration"
    )
    sf.write(out_path, audio, sr)
```

### 5.3. Voice changer pipeline

```python
def voice_changer(input_audio_path, target_speaker_embedding, out_path):
    # 1. Transcribe original audio
    transcript = transcribe_with_whisper(input_audio_path)
```

## 6. Multilingual output

Use a multilingual TTS model like **OpenAudio S1**, which is explicitly designed for expressive, multilingual TTS and cloning.

Key steps:

- **Language detection:** detect language from text or let user choose.  
- **Text normalization:** per‑language rules (numbers, dates, abbreviations).  
- **Model call:** pass `language_code` or `voice_locale` into the TTS model.

Conceptual:

```python
def tts_multilingual(text, speaker_embedding, lang, out_path):
    audio, sr = tts(
        model=multilingual_model,
        text=text,
        speaker_embedding=speaker_embedding,
        language=lang,  # "en", "es", "fr", etc.
        style="conversational"
    )
    sf.write(out_path, audio, sr)
```

OpenAudio supports uploading a short reference sample (5–10 seconds) and cloning that voice for TTS, including multilingual output.

## 7. API & app layer

### 7.1. REST API (FastAPI)

```python
from fastapi import FastAPI
from pydantic import BaseModel
import uuid

app = FastAPI()

class TTSRequest(BaseModel):
    voice_id: str
    text: str
    language: str = "en"
    style: str = "neutral"

@app.post("/tts")
def tts_endpoint(req: TTSRequest):
    job_id = str(uuid.uuid4())
    # enqueue background job: synthesize(req, job_id)
    return {"job_id": job_id}

@app.get("/tts/{job_id}")
def get_tts_result(job_id: str):
    # return URL or binary audio if ready
    return {"status": "completed", "audio_url": f"https://cdn/voices/{job_id}.wav"}
```

## 8. Production & safety considerations

- **Hardware:**  
  - Training: GPU (e.g., 1× RTX 4090 or cloud GPU).  
  - Inference: can be CPU for low volume, GPU for real‑time/scale.  
- **Caching:** cache outputs for repeated scripts (intros/outros).  
- **Rate limiting & auth:** protect your engine from abuse.  
- **Ethics & consent:** only clone voices you have explicit rights to; many open projects emphasize legal compliance.
